Before you start

Local site URL: http://127.0.0.1:8000 (or your server URL)
Start the app: Run php artisan serve from the project folder
Frontend: After code changes, run npm run build and hard-refresh the browser (Ctrl+F5)
Stripe testing: Add test keys in .env or Dashboard → Settings → Payment

1. Administrator testing

Log in as: admin@lmsacademy.local → you should land on Dashboard.

1.1 Users & learner types

  1. Go to Users List in the sidebar.
  2. Find an external user → open Edit → confirm you can set Employee or External.
  3. View a user’s CV (View CV link) if they uploaded one at registration.

1.2 Courses & audience

  1. Go to Courses → Manage Courses → open or create a course.
  2. Set Audience: Internal, Public, or Both.
  3. Set a course as Paid with a price for payment testing.
  4. Add a lesson (video, text, or document) and save.
  5. On a lesson resource, toggle Allow students to download on/off and save.

1.3 Student progress

  1. Go to Courses → Student Progress.
  2. Open a course → verify enrolled learners show completion %, quizzes, assignments.

1.4 Candidate pipeline (HR)

  1. Go to Candidate Pipeline.
  2. Open an external learner → check profile, CV, paid courses/exams, scores.
  3. Change status: New → In Review → Shortlisted → Hired (or Rejected).
  4. Add admin notes and save.
  5. Click View Payment History on the candidate profile.

1.5 Payments & refunds

  1. Go to Payment Report → Online Payments — confirm test payments appear.
  2. Go to Payment Report → Refund Tracking.
  3. Open a payment → set status to Refund Pending or Refunded with HR notes.
  4. On a Hired candidate, use Process Refund (Stripe sandbox) and confirm the dialog + result log.

1.6 Stripe setup (if checkout fails)

  1. Dashboard → Settings → Payment → Stripe → Active = ON, Test mode = ON.
  2. Enter test keys from Stripe Dashboard, or set STRIPE_KEY and STRIPE_SECRET in .env.

2. Trainer (instructor) testing

Log in as: trainer@lmsacademy.local → Dashboard with course tools (not full admin menus).

2.1 Course management

  1. Go to Courses → Manage Courses — see only your courses (not all admin courses).
  2. Create or edit a course: title, audience, pricing, lessons.
  3. Upload a video lesson and a PDF resource (test downloadable vs view-only).
  4. Publish/activate the course if required by your workflow.

2.2 Student progress (own courses only)

  1. Go to Courses → Student Progress.
  2. Confirm you see only your courses.
  3. Open a course → expand a student row → check completion, quiz pass/fail, assignments.

2.3 Trainer as learner

  1. Browse /courses/all as trainer (employee) → paid course shows Enroll, not Buy Now.
  2. Enroll in a paid course for free and open the course player.

2.4 What trainers should NOT see

  • Candidate Pipeline, Users List, Refund Tracking (admin-only areas).

3. Internal employee testing

Log in as: employee1@lmsacademy.local (or any @lmsacademy.local / @smartsourcingusa.com email).

3.1 Registration auto-detect (optional)

  1. Register with newemployee@lmsacademy.local → account should be Employee type automatically.

3.2 Catalog & internal courses

  1. Visit /courses/all while logged in.
  2. Confirm you see internal courses that guests cannot see (log out and compare).
  3. Open a paid public course → button should say Enroll (free), not Buy Now.
  4. Enroll and confirm the course appears under Student → My Courses.

3.3 Course player

  1. Open an enrolled course → start a video lesson.
  2. Try right-click and Ctrl+C in the player → should be blocked (content protection).
  3. Video should play (signed URL in network tab, not raw /storage/ link).
  4. Download a lesson resource marked downloadable; view-only resource shows eye icon only.
  5. Open assignment tab → download assignment PDF (should still work).

3.4 What employees should NOT see

  • Checkout/payment page for paid courses.
  • Candidate Pipeline (they are not in the hiring funnel).

4. External learner testing

Use: a personal email (not @lmsacademy.local or @smartsourcingusa.com).

4.1 Registration & catalog (guest)

  1. Log out → visit /courses/all as a guest.
  2. Confirm internal-only courses are not listed.
  3. Register at /register with external email, optional CV upload.
  4. Verify email if prompted, then log in.

4.2 Paid enrollment (Stripe sandbox)

  1. Open a paid course with audience Public or Both.
  2. Click Buy Now → complete checkout.
  3. Stripe test card: 4242 4242 4242 4242 · any future expiry · any CVC · any ZIP.
  4. After payment → redirected to course → enrollment confirmed under My Courses.

4.3 Free course

  1. Open a free public course → Enroll without payment.

4.4 Learning & progress

  1. Complete lessons, attempt quizzes, submit assignments.
  2. Check progress on the student course page.

4.5 Admin-side verification (switch to admin)

  1. Admin → Candidate Pipeline → external user appears with status New.
  2. Confirm paid enrollments, scores, and CV visible on candidate profile.
  3. Admin → Refund Tracking → payment shows as Paid.

5. End-to-end scenario (recommended demo)

Story: External learner buys a course → completes training → HR hires them → refund processed.

1. External user registers and buys a paid course (Stripe test card).
2. External user completes at least one lesson and one quiz.
3. Admin opens Candidate Pipeline → reviews CV and scores.
4. Admin sets status to Hired.
5. Admin clicks Process Refund → confirms → checks Refund Tracking + attempt log.
6. Employee user enrolls free in the same paid course → confirms no checkout.
